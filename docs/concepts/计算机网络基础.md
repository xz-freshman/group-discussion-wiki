# 计算机网络基础

## 概述

本部分讲解了计算机网络从物理层到应用层的基本原理，强调**层层叠加、向前兼容**是互联网发展的核心逻辑。

---

## 核心概念

### 1. MAC 地址

**定义：** 网卡物理地址，6 位十六进制数，写死在网卡上，前三位是制造商编号，后三位是产品编号，**全球唯一**。

**作用：** 在本地网络（同一网段）中精确寻址。所有机器都能收到广播消息，但 MAC 地址不匹配则丢弃。

**特点：** 不能伪造（写死在硬件里）。MAC 负责本地网络精确寻址，IP 负责子网间路由，两者各司其职。

### 2. 集线器（Hub）

多台机器通过集线器共享介质，信号发送到所有端口。引入 CSMA/CD 协议（载波监听多路访问）：发消息前先监听，忙则随机等待后重发。**效率低**，机器多了冲突严重。

### 3. 交换机（Switch）

维护 **MAC 表**（端口 ↔ MAC 地址），实现智能转发。学习过程：新机器发消息时先泛红（发送到所有端口），目标返回后学习到 MAC 地址。

- 交换机**本身没有 MAC 地址**（纯转发设备）
- MAC 表过大时需要**划分子网**来管理

> 📎 **补充知识①：ARP 协议**（讲座未涉及）
>
> 讲座提到交换机通过 MAC 表智能转发，但没讲设备最初如何通过 IP 找到目标 MAC 地址——这需要 **ARP（Address Resolution Protocol）**。
>
> 当 A 已知目标 IP 但不知道其 MAC 地址时，会在本地子网内广播 ARP 请求："谁是这个 IP？请告诉我你的 MAC 地址"。目标机器收到后返回 ARP 响应，交换机据此更新 MAC 表。
>
>ARP 表（ARP Cache）存的是 IP ↔ MAC 的映射，与交换机的 MAC 表类似，但属于操作系统层（网络层和数据链路层之间）。

### 4. 路由器（Router）与子网划分

- 路由器通常只有**两个端口**，每个端口有一张网卡（有自己的 MAC 地址）
- 连接两个子网：内部只需要知道对方网段的 MAC 地址，不必学习全球所有 MAC 地址
- **网关**：子网内所有流量汇合的路由器端口，通常 IP 设为 `.1`（如 192.168.1.1）
- 路由器分隔 LAN（内网）和 WAN（外网）

> 📎 **补充知识②：路由表匹配机制**（讲座涉及但未深入）
>
> 路由表的工作原理：当路由器收到一个数据包，目的 IP 是 D，路由器遍历路由表各条目，用子网掩码与 D 做按位与（AND）运算，得到目标网段。匹配最具体的条目（即最长前缀匹配，Longest Prefix Match）后转发。
>
> 如果没有匹配条目，路由器使用**默认路由**（0.0.0.0/0），将数据包发往上一级网关，一路向上直到有匹配。

### 5. IP 地址

| 属性 | 说明 |
|------|------|
| IPv4 | 32 位二进制，每 8 位转十进制点分（已于 2019 年耗尽） |
| IPv6 | 增加位数，解决地址枯竭问题 |
| 子网掩码 | 标志哪些位是子网号（255.255.255.0 = 前 24 位） |
| DHCP | 动态分配 IP 的协议，通常由路由器（默认网关）充当 DHCP 服务器 |

**同子网设备：** 前若干位相同，后面的位由 DHCP 动态分配。

**公网 IP vs 内网 IP：** 公网 IP 必须向运营商或上级机构申请；内网保留地址段（192.168.x.x、172.16–172.31.x.x、10.x.x.x）不能直接用于公网通信。

> 📎 **补充知识③：NAT 网络地址转换**（讲座未涉及）
>
> 讲座提到公网 IP 和内网 IP，但没有解释**内网 IP 如何通过一个公网 IP 上网**——这靠 **NAT（Network Address Translation）**。
>
> 工作原理：当内网设备向外网发请求时，路由器把数据包中的**源 IP（内网 IP）和源端口**替换成**自己的公网 IP 和一个新端口**，并记下这个映射关系。当外网响应返回时，路由器根据映射表反向替换，把数据送到正确的内网设备。
>
> NAT 相当于给内网设备提供了一个"代理"，让多个设备共享一个公网 IP 上网。这也解释了为什么内网设备直接暴露在公网上是不安全的——它们本身没有公网 IP。

### 6. DNS 域名解析

- 域名从**后往前**读：SJTU.EDU.CN → CN → EDU → SJTU → 子域
- 输入网址后，计算机先向 DNS 服务器查询对应 IP，再访问
- DNS 服务器全球有多个备份，交大 DNS 负责 EDU.CN 下的子域名

> 📎 **补充知识④：DNS 递归查询 vs 迭代查询**（讲座未涉及）
>
> 讲座只提到计算机会向 DNS 服务器查询，但没讲 DNS 查询的实际过程。完整的 DNS 解析分两类：
>
> - **递归查询**：客户端把查询完全委托给 DNS 服务器，由 DNS 服务器代为查询全程，最终返回 IP（客户端视角简单）
> - **迭代查询**：根服务器返回下一级权威服务器地址，本地 DNS 再去问下一级，一层层追问
>
> 实际互联网中，DNS 服务器通常同时扮演递归和迭代查询的角色。

### 7. HTTP vs HTTPS

- **HTTP**：明文传输，不安全
- **HTTPS**：加入 SSL/TLS 加密层

> 📎 **补充知识⑤：TCP/UDP 与网络分层的位置**（讲座未涉及）
>
> 讲座中"层层叠加"的逻辑里，HTTP/HTTPS 属于**应用层**（第 7 层），DNS 属于**应用层**，但在传输层使用 UDP（DNS）或 TCP（HTTP）。IP 属于**网络层**（第 3 层），MAC 属于**数据链路层**（第 2 层）。
>
> 这五层的完整结构（从上到下）：
> - 应用层：HTTP、DNS、SMTP
> - 传输层：TCP（可靠）、UDP（快）
> - 网络层：IP、ICMP（ping）
> - 数据链路层：MAC、ARP
> - 物理层：电信号、光信号
>
> 讲座的逻辑是沿着这个层次自下而上展开的，理解这个分层结构有助于理清为什么各层各司其职。

### 8. 无线网络安全

- 无线信号是**广播式**的，所有设备物理上都能收到
- 攻击者可打开监听模式抓取所有信息（类似集线器时代的问题）
- 有线网络靠物理端口隔离；无线网络靠加密（WPA/WPA2）和上层协议（HTTPS）

> 📎 **补充知识⑥：Wi-Fi 的 AP 模式与监听模式**（讲座未涉及）
>
> 讲座提到无线网是广播式的，所有人都能收到。实际上无线网卡有几种工作模式：
> - **Managed 模式**（默认）：只接收发给自己的数据帧
> - **Monitor 模式**：接收所有无线空中的帧（相当于有线网络的 promiscuous mode），常用于网络诊断和渗透测试
> - **Ad-hoc 模式**：设备点对点直连，无需 AP
>
> 家庭路由器工作在 AP（Access Point）模式下，通过 SSID 和密码（WPA2）限制接入。开放网络不加密，所有数据明文传输，极其危险。

### 9. traceroute 与网络测量

追踪数据包经过的路由器节点，跨洋访问通过**物理海缆**连接（不是纯无线）。

## 关键设计思想

1. **层层叠加**：每一层新技术都是在旧技术基础上加上新功能，不是替换（MAC → Hub → Switch → Router → Internet）
2. **向前兼容**：MAC 层已完善，IP 层在其基础上叠加；IP 层已完善，再叠加 DNS、应用层
3. **子网划分**：减少每台设备需要维护的路由表大小
4. **广播与单播**：Hub 时代广播 → 交换机时代精确单播

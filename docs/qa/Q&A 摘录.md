# Q&A 摘录

以下是从第七周小组讨论中整理出的有价值提问与回答，按主题分类。

## 计算机网络部分

### Q：为什么还需要 MAC 地址，直接用 IP 不就行了？
**A：** IP 是分配到子网的，而 MAC 固定在机器上且不能重复。通信是一层层叠加的，必须向前兼容。MAC 层负责本地网络精确寻址，IP 层负责子网间路由。两者各司其职。

### Q：IP 地址可以伪造吗？
**A：** 可以（静态 IP 手动配置即可）。这导致纯用 IP 做身份认证不可靠，所以还需要 MAC 地址做双重保障。

### Q：为什么无线网络是危险的？
**A：** 无线信号是广播式的，所有设备物理上都能收到。默认网卡会丢弃非目标信号，但攻击者可以打开监听模式（promiscuous mode）抓取所有信息。有线网络靠物理端口隔离；无线网络靠加密协议和上层安全机制保护。

### Q：中国防火墙部署在哪？
**A：** 大概部署在三大运营商出口。代理服务器则是在发送请求时先经过代理，由代理转发到国外。

### Q：traceroute 追踪的是什么？
**A：** 数据包从本机到目标地址沿途经过的路由器节点。可以了解网络的物理拓扑结构，跨洋访问通过海缆（物理连接），不是纯无线。

## 宇宙学部分

### Q：为什么可观测宇宙半径不是 C/H₀ ≈ 144 亿光年？
**A：** 因为宇宙膨胀历史不均匀。早期膨胀更快，光在很久以前走过的距离后来被空间膨胀拉长了（相当于走了"更多的路"）。真实可观测半径约 480 亿光年（直径 960 亿光年）。

### Q：宇宙微波背景辐射（CMB）的红移是多少？
**A：** Z ≈ 1100。对应宇宙在约 38 万年、光子与电子解耦合时期（复合纪元）的状态。太阳温度约 6000K，CMB 约 3K，红移 ≈ 6000/3 ≈ 1100。

### Q：FRW 度规中时间 t 的选取有什么物理意义？
**A：** 任意与宇宙共动的参考系中测得的 a(t) 都是相同的（均匀性保证了这一点）。选不同参考系只改变时间单位定义，不改变 a(t) 的函数形式。时间参数选取是任意的。

### Q：暴涨（Inflation）解决了什么问题？
**A：** 解释了为什么宇宙是平坦的（K ≈ 0）。早期宇宙经历极端指数膨胀，把任何初始曲率都"拉平"了。同时也解释了视界问题和平坦性问题的起源。

### Q：为什么天文实验测得的误差那么大却还能用？
**A：** 天文实验以量取胜：宇宙中星系数量极大（n → ∞），统计误差 ∝ 1/√n。只要 n 足够大，误差可以压到很小。这是天文观测与粒子物理实验的本质区别。

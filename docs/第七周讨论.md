## 第一部分：计算机网络基础知识

### 原始通信：一台机器 A 和机器 B

最原始的形态是两台机器直接用双绞线连接。信号通过电线发送，A 和 B 都能收到。

### MAC 地址

- MAC 地址（物理地址）是 6 位十六进制数，写死在网卡上
- 前三位是制造商编号，后三位是网卡内部产品编号，全球唯一
- 当 A 向 B 发消息时，信号携带目标 MAC 地址，所有机器都会收到，但 MAC 地址不匹配的机器会丢弃消息

### 集线器（Hub）的问题

- 多台机器用集线器连接：所有信号发送到所有端口
- 载波监听多路访问（CSMA）：发消息前先监听线路，忙则等待随机时间重发
- 机器一多效率低下

### 交换机（Switch）

- 交换机维护一张 MAC 表（端口 → MAC 地址）
- A 发消息给 D：交换机查 MAC 表 → 从对应端口发出
- 学习过程：新消息泛红 → 目标返回 → 学习到对方 MAC 地址
- 交换机没有 MAC 地址（纯转发设备）
- MAC 表过大时需要划分子网

### 子网划分与路由器

- 路由器通常只有两个端口，每个端口有一张网卡（有自己的 MAC 地址）
- 路由器分隔两个子网，内部只需要知道对方网段的 MAC 地址
- 每台机器只需维护本子网的路由表，不必学习全球所有 MAC 地址

### IP 地址

- IPv4：32 位二进制数，每 8 位转十进制，用点分表示（如 192.168.1.1）
- IPv4 地址已于 2019 年耗尽；IPv6 增加位数以解决地址不足
- 子网掩码：二进制中哪些位是子网 IP（255.255.255.0 → 前 24 位是子网）
- 同子网内设备前若干位相同，后面的位由 DHCP 动态分配
- 路由器本身占 1（如 192.168.1.1），称为默认网关

### 无线网络

- 无线网卡在物理层上开辟虚拟端口
- 无线网是广播式的，所有设备都能收到所有信息（相对不安全）
- 路由器通过端口区分设备；无线无法做到点对点，靠虚拟端口映射

### 公网 IP vs 内网 IP

- 内网（LAN）：路由器以内的局域网，IP 由 DHCP 分配
- 外网（WAN）：连接到上级路由器
- 公网 IP 必须向运营商或上级机构申请，不能随意指定
- 判断方法：看 IP 所属的地址段是否属于保留的内网段（如 192.168.x.x、172.16–172.31.x.x、10.x.x.x）

### 域名解析（DNS）

- 输入域名后，计算机会先向 DNS 服务器查询对应的 IP
- DNS 服务器全球有多个备份
- 域名从后往前读：SJTU.EDU.CN → CN → EDU → SJTU → PANAX → ...
- 交大 DNS 负责 EDU.CN 下的子域名

### HTTP vs HTTPS

- HTTP：明文传输，不加密
- HTTPS：加入 SSL/TLS 加密层

### traceroute

- `traceroute`（Windows 是 `tracert`）：追踪数据包经过的路由器节点
- 跨洋访问通过海缆物理连接，不是纯无线

### 中国防火墙与代理服务器

- 防火墙大概部署在三大运营商出口
- 代理服务器：请求先发到代理，再由代理转发到国外

### 动态 IP vs 静态 IP

- DHCP：动态主机配置协议，由路由器（默认网关）分配 IP
- 静态 IP：手动配置（理论上 IP 可以伪造）

## 第二部分：宇宙学基础知识

### 可观测宇宙的半径

- 错误理解：可观测宇宙 = C/H₀ ≈ 144 亿光年（半径）
- 正确理解：考虑宇宙膨胀历史后，可观测宇宙半径约 **480 亿光年**（直径约 960 亿光年）
- 原因：光在很久以前走过的距离后来被空间膨胀拉长了

### 宇宙学红移（Cosmological Redshift）vs 多普勒效应

- 初学者常把红移解释为简单的多普勒效应
- 实际上：红移是由于空间本身膨胀导致光波长被拉长（不同于多普勒）
- 广义相对论处理的是度规，不是简单的速度叠加

### Scale Factor a(t)

- 规定今天 a = 1
- 宇宙中物理距离 = a(t) × R₀（R₀ 是共动距离，一个固定标尺）
- a(t) 随时间变化：早期增长快 → 物质主导期放缓 → 暗能量主导期再次加速（指数增长）
- 这个 a(t) 的演化历史可以从 FRW 度规结合宇宙学参数推出

### FRW 度规（Friedmann–Robertson–Walker Metric）

- 核心假设：宇宙是**均匀各向同性**的（Homogeneous & Isotropic）
- 所以度规只能写为只关于时间的函数，空间部分各向同性
- 度规形式：`ds² = -c²dt² + a(t)² · (dr² + r²dΩ²)` （简化版）
- FRW 方程：从度规出发结合爱因斯坦场方程可以推导宇宙膨胀的动力学方程

### 宇宙学标准模型的三种能量组分

| 组分 | 符号 | 随 a(t) 稀释的方式 |
|------|------|-------------------|
| 辐射（Radiation） | Ω_R | ∝ a⁻⁴（光子能量∝a⁻¹，数密度∝a⁻³）|
| 物质（Matter） | Ω_M | ∝ a⁻³（仅数密度稀释）|
| 暗能量（Dark Energy） | Ω_Λ | ∝ 常数（能量密度不变）|

### 暗能量的物理意义

- 暗能量密度不随宇宙膨胀变化（类似真空能）
- 宇宙膨胀的功最终来自暗能量的转化（能量守恒不适用于整体宇宙）
- 当前宇宙由暗能量主导（Ω_Λ ≈ 70%）

### 宇宙的 future：指数增长

- 在暗能量主导下，a(t) 呈指数增长（exponential growth）
- 未来尺度因子会非常大

### CMB（宇宙微波背景辐射）

- CMB 光子与电子解耦合时期（复合纪元）发出的光
- 温度 ≈ 3 K（当前观测值）
- 太阳表面温度 ≈ 6000 K
- 红移估算：Z ≈ T_sun / T_CMB ≈ 6000/3 ≈ 1100（实际约 1100）
- 观测 CMB 相当于看到宇宙在约 38 万年时的状态

### Hubble Tension（哈勃张力）

- **方法一**：测近邻宇宙（低红移）距离-红移关系，代入标准模型拟合 → H₀ ≈ 73
- **方法二**：测 CMB（高红移）用标准模型参数拟合 → H₀ ≈ 67
- 两者均为小误差，但不一致 → 系统性偏差，可能是模型不完整或新物理
- 目前宇宙学最大的难题之一

### 宇宙曲率 K

- 当前观测表明宇宙是平坦的（K ≈ 0）
- 但为什么恰好接近零？暴涨理论（Inflation）可以解释

### 天文观测方法的特点

- 天文实验误差大（以量取胜：n 个星系 → 误差 ∝ 1/√n）
- 与粒子物理不同，天文实验无法在受控条件下重复
- 宇宙中星系数量极大 → 统计误差可以压得很小

### 关于物理常数的测定

- 有些常数测得很准（基于单一实验）
- 有些常数（如 H₀）测不准，不同方法给出不同结果
- 标准模型中的参数（Ω_R, Ω_M, Ω_Λ）由多种观测联合拟合得出

## 第三部分：精彩问答摘录

### Q1：为什么还需要 MAC 地址，直接用 IP 不就行了？
**A：** IP 是分配到子网的，而 MAC 固定在机器上且不能重复。通信是一层层叠加的，必须向前兼容。MAC 层负责本地网络精确寻址，IP 层负责子网间路由。

### Q2：IP 地址可以伪造吗？
**A：** 可以（静态 IP 手动配置即可）。这导致纯用 IP 做身份认证不可靠，所以还需要 MAC 地址做双重保障。

### Q3：无线网络为什么是危险的？
**A：** 无线信号是广播式的，物理上所有人都能收到，只是默认网卡会丢弃非目标信号。攻击者可以打开监听模式（promiscuous mode）抓取所有信息。

### Q4：为什么可观测宇宙半径不是 C/H₀ ≈ 144 亿光年？
**A：** 因为宇宙膨胀历史不均匀。早期膨胀更快，光在早期走过的距离被后来空间膨胀拉长了，相当于走了"更多的路"。真实可观测半径是 480 亿光年。

### Q5：宇宙微波背景辐射（CMB）的红移是多少？
**A：** Z ≈ 1100。对应的是宇宙在约 38 万年、光子与电子解耦合时的状态。

### Q6：FRW 度规中时间 t 的选取有什么物理意义？
**A：** 任意与宇宙共动的参考系中测得的 a(t) 都是相同的（均匀性保证了这一点）。选不同的参考系只会改变时间单位的定义，不改变 a(t) 的函数形式。

### Q7：暴涨（Inflation）解决了什么问题？
**A：** 解释了为什么宇宙是平坦的（K ≈ 0）。早期宇宙经历极端指数膨胀，把任何初始曲率都"拉平"了。同时也解释了视界问题和平坦性问题的起源。
